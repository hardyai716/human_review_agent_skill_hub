# 人审运营 SOP 四节点填表稿

本文沉淀运营 SOP 在 Agent + Skill 架构下的填写口径，按“感知、分析、触达、解决”四个节点描述每个场景应该做什么。

适用表格列：

```text
分类 / 事项 / 指标 / 感知 / 分析 / 触达 / 解决 / 数据来源 / Skill 能力
```

填写原则：

- 感知：识别场景、指标、阈值、时间窗口、维度、数据源和数据就绪状态。
- 分析：查数、计算指标、命中规则、拆解维度、归因、输出证据和建议动作。
- 触达：定位责任人，生成通知文本、卡片、报表和发送计划；真实发送前保留确认。
- 解决：记录动作、证据、复查结果和最终结论；满足闭环条件后关闭。

## 1. 效率-队列设计合理性

目标：不合理队列数归 0。

判定口径：命中任一不合理规则即判定为不合理队列，统计时按队列去重。当前重点规则包括同风险同体裁多队列、同风险跨体裁多队列、无效队列流转、优先级/废弃/兜底逻辑不合理、长尾/无人认领队列。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别效率-队列设计合理性场景，读取风险类型、送审体裁、送审队列、队列流转、优先级、废弃逻辑、兜底逻辑、长周期有效进审等信息；抽取风险类型、体裁、队列、流转链路、进审量、审核量、有效审核量、时间窗口和配置规则。 |
| 分析 | 按业务逻辑合理性和队列设计合理性判断不合理队列：包括不该建队列、同风险同体裁多队列、同风险跨体裁多队列、无效/循环流转、优先级/废弃/兜底黑盒、非底线待审阻塞、长尾/无人认领队列等；任一规则命中即判定为不合理队列，并按队列去重统计。 |
| 触达 | 触达人审运营同学，必要时同步治理 BP；通知内容包含不合理队列清单、命中规则、风险类型、体裁、数据证据、影响说明和建议动作。下线、合并、改流转、改优先级、改废弃逻辑等动作必须人工确认，不自动修改配置。 |
| 解决 | 记录处理动作和结论，包括不建/撤销队列、下线队列、合并队列、调整体裁、修改流转规则、补充触发/终止条件、调整优先级/废弃/兜底逻辑等；复查不合理队列数是否归 0，并沉淀配置变更记录、证据和最终闭环结论。 |

| 运营场景 | Skill 描述 | 输入 | 输出 |
| --- | --- | --- | --- |

## 2. 效率-打标率低效治理

目标：低效策略治理，打标率提升或策略下线。

指标口径：打标率 = 打标量 / 审核完结量。低效策略定义为周维度 review 策略打标率 `<10%`，且进审量大于 0。治理要求：`<5%` 建议下线，`5%-10%` 要求提升到 10% 以上；连续 2 周未达标或未下线，升级至 +1。这里的 `<10%` / `<5%` 是治理目标和动作阈值，`notice/P2/P1/P0` 是通知严重等级，按低效程度、持续性、日均进审量和风险域爆量等规则判定，不能做成阈值到等级的一一映射。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别效率-打标率低效治理场景，抽取打标率、低效阈值 `<10%`、进审量大于 0、周维度时间窗口，以及策略 ID、策略名称、风险域、机审一级标签、可选 reason、POC 等维度；读取指标口径和数据集说明，并区分治理阈值与通知严重等级。 |
| 分析 | 查询指定周期和对比周期的进审量、完审量、打标量、打标率；默认按机审一级标签 × 策略 ID × 策略名称筛选低效策略，并将风险域爆量规则按机审一级标签汇总低效策略后评估；输出低效策略清单、影响量级、通知等级证据和治理建议，禁止把 `<10%` / `<5%` 直接等同为 `notice/P2/P1/P0`。 |
| 触达 | 按策略、风险域标签与 POC 映射生成责任人；风险域维度行策略 ID/名称为空但仍按机审一级标签路由；生成治理通知、卡片、报表和发送计划；通知等级 `notice/P2/P1/P0` 只表达触达严重程度，治理动作仍按 `<10%` / `<5%` 目标推进；确认目标群、收件人、发送身份和内容后触达。 |
| 解决 | 记录 POC 反馈、治理动作、DDL 和证据；按 DDL 回收打标率效果，确认策略是否下线或提升至 10% 以上；连续 2 周未达标或未下线则升级至 +1，满足动作、证据、结论后闭环。 |

## 3. 效率-自动处置准确率

目标：整体自动处置准确率达到 80%，特殊风险域可与业务负责人对齐差异化目标。

指标口径：自动处置准确率 = 三级标签处置准确量 / 总自动处置评估量。预警规则：日维度准确率较前周期下降 3pp 及以上；周维度机审日均处置量 50+ 且准确率小于 30%。日维度预警要求 3 天内提升，周维度低准策略要求 1 周内提准或下线，逾期升级治理 BP +1/+2。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别效率-自动处置准确率场景，不误触发打标率；抽取自动处置准确率、整体目标 80%、特殊风险域差异化目标、日/周时间窗口、风险域、策略、三级标签等维度；读取指标口径、数据集说明和治理 BP / 策略 POC 映射。 |
| 分析 | 查询自动处置量、总评估量、处置准确量、错误处置量和准确率；按整体、风险域、策略、三级标签拆解；识别日维度下降 3pp 及以上、周维度日均处置量 50+ 且准确率小于 30% 的低准对象，输出清单、证据、影响量级和建议动作。 |
| 触达 | 按风险域、策略、三级标签映射治理 BP / 策略 POC；生成预警或治理通知、卡片、报表和发送计划；日维度要求 3 天内提升，周维度要求 1 周内提准或下线；确认目标群、收件人和内容后触达。 |
| 解决 | 记录提准、下线、优化动作、DDL 和证据；按 DDL 回收准确率效果，确认是否达标或下线；逾期未提升/未下线则升级治理 BP +1/+2；满足动作、证据、结论和复查结果后闭环。 |

## 4. 效率-审核延时/爆量

目标：队列体感延时达标率 100%，差异化目标按体裁、风险程度、待审状态与业务对齐。

指标口径：体感延时 = 用户投稿时间到任务审核结束时间。子类型包括涉政延时、非涉政延时；原因拆分包括召回延时、人审延时、上游送审超量、下游人力不足、审核效率下降、调度不足。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别效率-审核延时/爆量场景，区分涉政/非涉政、召回延时/人审延时；抽取体感延时达标率、目标值、队列、体裁、风险域、待审状态和时间窗口；确认队列延时口径、数据源和差异化目标。 |
| 分析 | 查询各队列进审量、审核量、待审量、超时量、达标率、最大延时、P95/P99 延时；识别未达标队列、超时集中时段、积压量超预期队列和相关策略；定位上游召回异常、送审爆量、下游人力不足或效率问题。 |
| 触达 | 按队列 Owner、值班 Owner、治理 BP、CQC 调度负责人等生成触达计划；通知未达标队列、超时量、影响时段、原因判断和建议动作；召回/平台问题触达研发，上游爆量触达治理，下游人力问题触达 CQC 调度，并按 1h/2h 规则升级。 |
| 解决 | 记录积压队列、责任 POC、处理动作和恢复进度；定时轮询延时达标率和积压变化，确认队列是否恢复常规状态；未缓解按规则升级；关闭前记录原因、动作、证据、复查结果，P0/P1 体验事故需 24h 复盘、1 周内沉淀系统方案。 |

## 5. 质量-质检准确率

目标：整体质检准确率 95% 以上，涉政底线要求 98%。

指标口径：质检准确率 = 质检准确量 / 总质检量；通过准确率 = 通过准确量 / 通过质检量；打标准确率 = 打标准确量 / 打标质检量。质检方式包括随机质检、交叉质检；8 人及以上队列优先交叉质检，小于 8 人队列随机质检；质检方式需为盲检。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别质量模块-质检准确率场景，抽取质检准确率、通过准确率、打标准确率、目标阈值、日/周时间窗口、风险域、队列、质检方式和供应商模式；读取质检数据集、指标口径、质检量置信规则和责任人映射。 |
| 分析 | 查询质检量、准确量、通过/打标质检量及准确量，计算整体、风险域、通过、打标准确率；判断是否低于整体 95% 或涉政底线 98%，并校验质检量是否满足置信要求；输出统计结果、明细和波动归因。 |
| 触达 | 按风险域、队列、CQC 群组和人审运营映射生成触达对象；单周不达标触达群组质量负责人和人审运营，连续三周不达标升级 CQC 总负责人、人审运营负责人；通知包含指标差距、原因和整改 DDL。 |
| 解决 | 记录 T+1 纠偏、SOP 优化、培训、补充质检等动作；1 周内复查质检准确率和投放准确率是否提升；连续不达标则升级并持续跟进，满足动作、证据、复查结果和结论后闭环。 |

## 6. 质量-漏放事故监控

目标：S01 底线事故人审 0 漏放，S23 非底线事故下降。当前为线下数据。

核心指标：S01 人审漏放事故数、S23 非底线事故数、事故投放验收漏放率、事故定级准确率。S01 事故投放验收漏放率要求为 0，事故定级准确率要求 95% 以上。每个人审底线事故漏放需 24h 内输出复盘报告，事故发生 1 周内完成投放验收。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别质量模块-漏放事故监控场景，读取线下事故台账、事故定级结果、复盘记录、投放验收结果和周度质检记录；抽取事故等级、风险域、人审漏放标记、复盘状态、验收状态和责任人映射。 |
| 分析 | 统计 S01 人审漏放事故数、S23 非底线事故数、事故投放验收漏放率和事故定级准确率；识别 S01 未清零、S23 未下降、1 周验收未达成、定级准确率小于 95% 或存在 miss 风险的对象。 |
| 触达 | 按事故等级、风险域、CQC 群组和人审运营映射生成触达对象；S23 或 miss 风险触达 CQC 群组负责人及人审运营负责人，明显 S01 底线事故漏放升级 CQC 总负责人。 |
| 解决 | 记录 24h 复盘、整改动作、投放验收、标准优化和复查结果；S01 漏放率未归 0 则持续复投，事故定级准确率不足则推动标准优化；满足复盘、验收、整改、证据和结论后闭环。 |

## 7. 质量-申诉成功率

目标：申诉成功率小于 10%。

默认统计对象：作者。指标口径：申诉成功率 = 申诉成功作者量 / 申诉作者量。若后续按视频或账号统计，需替换分子分母对象。预警升级：P2 风险域单周不达标，P1 风险域双周不达标，P0 风险域三周不达标。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别质量-申诉成功率场景，确认统计对象为视频/账号/作者，默认按作者口径计算；抽取目标小于 10%、日/周时间窗口、风险域、标签、策略、申诉原因等维度；读取申诉数据集、指标口径和责任人映射。 |
| 分析 | 查询申诉作者量、申诉成功作者量、申诉成功率；按风险域、标签、策略、申诉原因拆解；识别单周/双周/三周不达标风险域，并对申诉成功案例做原因聚类，输出异常清单、原因占比、可优化空间和治理建议。 |
| 触达 | 按风险域和责任人映射触达治理 BP、人审运营及对应 +1/+2；P2 单周不达标，P1 双周不达标，P0 三周不达标；通知包含目标差距、连续不达标周数、归因摘要、建议动作和整改 DDL。 |
| 解决 | 记录机审提准、人审纠偏、标准优化、case 补充、申诉执行口径对齐等动作；按 DDL 回收申诉成功率变化，验证是否降至目标或阶段性目标；持续不达标则升级，满足动作、证据、复查结果和结论后闭环。 |

## 8. 质量-质量投放

目标：底线风险投放准确率 100%，其他风险投放准确率 95% 以上。当前为线下数据。

指标口径：投放准确率 = 审核员准确审出 case 量 / 总投放量。投放对象包括事故、罚单、指令、LS 重保节点、暑期等特殊时期风险 case。投放到队列中的审核结果不生效，2-3 天后回收投放准确率。不达标需复投。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别质量-质量投放场景，读取线下投放台账、投放 case 明细、队列投放记录和回收结果；抽取投放批次、风险类型、是否底线风险、队列、总投放量、准确审出量、初投/复投状态和责任人映射。 |
| 分析 | 计算投放准确率 = 审核员准确审出 case 量 / 总投放量；判断底线风险是否达到 100%、其他风险是否达到 95% 以上；识别单次不达标、复投不达标、连续初投/复投不达标，并输出批次清单、错误 case、差距和复投建议。 |
| 触达 | 按 CQC 群组和人审运营映射触达责任人；P2 单次投放不达标触达 CQC 群组质量负责人、人审运营，P1 单次复投不达标或连续两次初投不达标升级 +1，P0 连续两次复投或连续三次初投不达标升级 CQC 总负责人、人审运营 +2。 |
| 解决 | 记录复投、纠偏、培训、标准澄清、SOP 优化等动作；回收复投准确率，底线风险需达 100%，其他风险达 95% 以上；未达标继续复投并按规则升级，满足动作、证据、复投结果和结论后闭环。 |

## 9. 质量-二确 diff 率

目标：参考申诉 diff 率测算逻辑，与业务对齐阶段性目标。

指标口径：二确 diff 率 = 二确环节打标与前置队列打标 diff 量 / 总流转二确量。监控周期为日度监控和周度波动预警。预警升级：P2 风险域单周波动大于 10%，P1 大于 20%，P0 大于 30%。注：需后续明确波动是相对涨幅还是百分点变化。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 识别质量-二确 diff 率监控场景，抽取二确 diff 率、日/周时间窗口、风险域、标签、策略、前置队列、二确队列和 diff 原因；读取二确流转数据、前置队列打标结果、二确打标结果、指标口径和责任人映射。 |
| 分析 | 计算二确 diff 率 = 二确环节打标与前置队列打标 diff 量 / 总流转二确量；按风险域、标签、策略、队列拆解；识别单周波动大于 10% / 20% / 30% 的对象，并对 diff case 做聚类归因，输出异常清单、原因占比和优化建议。 |
| 触达 | 按风险域、标签、策略映射治理 BP 和人审运营；P2 单周波动大于 10%、P1 大于 20%、P0 大于 30% 生成预警通知，内容包含波动幅度、异常对象、归因摘要、典型 case、建议动作和整改 DDL。 |
| 解决 | 记录机审提准、人审纠偏、标准尺度对齐、标签边界澄清、case 补充、SOP 优化等动作；按 DDL 回收二确 diff 率变化，验证是否回落；持续高波动则升级跟进，满足动作、证据、复查结果和结论后闭环。 |
