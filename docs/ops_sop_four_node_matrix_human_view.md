# 人审运营 SOP 四节点填表稿（人视角）

本文是 `docs/ops_sop_four_node_matrix.md` 的对照版本。前一版从 Agent 视角出发，描述“Agent 应该做什么”；本版从人的视角出发，描述“每个节点产出什么、需要什么条件、触发什么动作”。

格式：

```text
节点名称：
一行概括

- 动作 / 条件 / 维度（用 "-" 分条）
- 输出材料 / 触发规则 / 闭环条件（用 "-" 分条）
```

## 1. 效率-打标率低效治理

SOP：
1. 低效策略定义：打标率（打标量 / 审核完结量）< 10% 的送审策略。
2. 低效策略分析：周维度 review 策略打标率低于 10% 的策略（逐步提升要求），分析策略归属风险域、进审量、打标率。
3. 定要求：要求治理 POC by 策略给出治理优化预期和节奏（`<5%` 建议下线、`5%-10%` 提升到 10% 以上），定期 review 治理进度。这里的 `<10%` / `<5%` 是治理目标和动作阈值，不是 `notice/P2/P1/P0` 通知等级。
4. 追进展，回收效果：根据与治理对齐的 DDL 回收优化效果，针对没有达标的策略需给出优化动作和 DDL，如持续 2 周依然未达标或未下线，需上升至 +1。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 按照周维度时间窗口，抽取策略打标率相关数据<br><br>- 数据维度：打标率、审核量、风险域、机审一级标签、策略 ID、策略名称、可选 reason、策略 owner<br>- 数据就绪：确认指定周期数据已产出、审核量 > 0、维度字段无缺失<br>- 口径分层：`<10%` / `<5%` 是治理阈值，`notice/P2/P1/P0` 是通知严重等级 |
| 分析 | 分析输出低效策略清单<br><br>- 低效阈值：筛选打标率 `<10%`、审核量 > 0 的策略，默认按机审一级标签 × 策略 ID × 策略名称聚合；风险域爆量规则先筛低效策略再按机审一级标签汇总<br>- 治理建议：打标率 `<5%` 建议下线，`5%-10%` 建议提升至 10% 以上<br>- 通知等级：`notice/P2/P1/P0` 按低效程度、持续性、日均进审量和风险域爆量规则判定，不与 `<10%` / `<5%` 一一映射<br>- 输出材料：低效策略清单、影响量级、影响周期、通知等级证据和治理建议 |
| 通知 | 飞书通知策略 owner，触发定级条件时上升沟通<br><br>- 根据低效策略在飞书群聊、私聊通知到策略 owner<br>- `notice/P2/P1/P0` 只表达触达严重程度和升级范围，治理动作仍按 `<10%` / `<5%` 目标推进<br>- 要求 owner 给出优化预期、治理动作和 DDL<br>- 连续 2 周未达标或未下线，上升至策略 owner 的 +1 |
| 解决 | 回收策略打标率结果，确保闭环解决<br><br>- 记录策略 owner 反馈、治理动作、DDL 和证据<br>- 按 DDL 回收打标率优化效果<br>- 确认策略状态为以下之一：1. 下线 / 2. 提升至 10% 以上 / 3. 治理 +1 评估保持不变<br>- 满足上述三个条件后闭环解决 |

## 2. 效率-自动处置准确率

SOP：
1. 数据口径：三级标签准确率 = 三级标签处置准确量 / 总自动处置评估量。
2. 制定目标：与治理 BP 对齐目标，整体准确率 80%，特殊风险域可制定差异化目标。
3. 数据监控：日度 / 周度整体准确率、风险域准确率、三级标签准确率。
4. 预警 & 升级：日维度准确率降低 3pp 及以上并预警治理 BP，要求 3 天内提升；周度拉取机审日均处置量 50+ 且准确率 < 30% 的策略，推动治理 BP 一周内提准或下线；如逾期未达成则升级给治理 BP 的 +1/+2。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 按照日 / 周时间窗口，抽取自动处置准确率相关数据<br><br>- 数据维度：自动处置准确率、总自动处置评估量、三级标签处置准确量、风险域、策略 ID、三级标签、策略 owner<br>- 数据就绪：确认日度 / 周度数据已产出、评估量满足最低样本要求<br>- 特殊规则：涉政 / 指令舆情等特殊风险域有差异化目标，需单独确认 |
| 分析 | 分析输出低准策略清单和影响评估<br><br>- 日度监控：识别准确率较前周期下降 3pp 及以上的风险域 / 策略<br>- 周度监控：筛选机审日均处置量 50+ 且准确率 < 30% 的低准策略<br>- 维度拆解：按整体、风险域、策略、三级标签拆解下降来源<br>- 输出材料：不及预期清单、影响量级、下降幅度、低准原因和治理建议 |
| 通知 | 飞书通知治理 BP 和策略 owner，触发升级条件时上升沟通<br><br>- 日维度准确率下降 3pp 及以上，通知治理 BP，要求 3 天内提升<br>- 周维度日均处置量 50+ 且准确率 < 30%，通知策略 owner，要求 1 周内提准或下线<br>- 逾期未达成，上升至治理 BP 的 +1/+2 |
| 解决 | 回收自动处置准确率结果，确保闭环解决<br><br>- 记录提准动作、下线动作、优化动作、DDL 和证据<br>- 按 DDL 回收准确率效果<br>- 确认策略状态为以下之一：1. 提准至 80% 或差异化目标 / 2. 下线 / 3. 治理 BP +1/+2 评估保持不变<br>- 满足上述三个条件后闭环解决 |

## 3. 效率-审核延时 / 爆量

SOP：
1. 数据口径：体感延时 = 用户投稿时间到任务审核结束时间，区分涉政延时和非涉政延时。
2. 目标：队列体感延时达标率 100%，不同体裁、风险程度、待审状态可有差异化目标。
3. 延时监控 & 预警：实时 by 队列监控体感延时，包括召回延时和人审延时；召回延时一经发现立刻反馈研发；人审延时区分上游送审超量和下游人力不足。
4. 升级机制：实时延时超 1h 上升至治理负责人 / 人审运营负责人；上游爆量需治理 10min 内响应，持续上涨 cc +1；下游人力问题需 CQC 调度加人，持续 2h 未缓解上升至 CQC 总负责人。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 实时监控队列延时数据，区分延时类型<br><br>- 数据维度：体感延时达标率、队列 ID、进审量、待审量、超时量、体裁、风险域、涉政 / 非涉政标记<br>- 延时类型：召回延时（上游事故、模型重试失败、平台不可用）、人审延时（上游送审超量、下游人力不足、审核效率下降）<br>- 数据就绪：确认实时监控数据可获取、队列 Owner 和值班 Owner 映射已配置 |
| 分析 | 分析输出延时异常队列清单和积压原因<br><br>- 延时指标：各队列进审量、审核量、待审量、超时量、达标率、最大延时、P95/P99 延时<br>- 异常识别：未达标队列、超时集中时段、积压量超预期队列<br>- 原因定位：上游召回异常、上游送审爆量、下游人力不足或审核效率下降<br>- 输出材料：延时异常清单、积压量、影响范围、原因判断和升级建议 |
| 通知 | 按延时原因分路径通知对应责任人，触发升级条件时上升沟通<br><br>- 召回延时 / 平台不可用：立即反馈研发处理；影响审核员停审或 VOC P2 定级时上报治理负责人、研发负责人<br>- 上游送审爆量：通知治理 BP，要求 10min 内响应、30min 观测效果；持续上涨则二次通知并 cc +1、人审爆量运营、审核 VOC POC<br>- 下游人力不足：通知 CQC 调度负责人加大人力<br>- 实时延时超 1h：上升至治理负责人、人审运营负责人；持续 2h 未缓解：上升至群组负责人 / CQC 总负责人 |
| 解决 | 跟踪队列延时恢复，确保闭环解决<br><br>- 记录积压队列、责任 POC、处理动作和恢复进度<br>- 定时轮询延时达标率和积压变化，确认队列恢复常规状态<br>- P0/P1 体验事故需 24h 内完成复盘，1 周内提出系统化能力建设需求<br>- 未缓解按规则升级；关闭前记录原因、动作、证据和复查结果 |

## 4. 效率-队列设计合理性

SOP：
1. 业务逻辑合理性：队列不是所有业务的标配，引入队列前需评估风险程度、机审能力、审核体裁。
2. 队列设计合理性：解决同风险同体裁多队列、同风险跨体裁多队列、无效队列流转、优先级 / 废弃 / 兜底不合理、待审阻塞、长尾无人认领队列等问题。
3. 目标：不合理队列数归 0。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 抽取队列配置和进审数据，覆盖三类巡检规则<br><br>- 数据维度：风险类型、送审体裁、队列 ID、队列名称、进审量、审核量、有效审核量、流转链路、优先级配置、废弃配置、兜底配置、队列 Owner、公开周知状态<br>- 巡检规则：体裁不一、队列不一、长周期无有效进审<br>- 数据就绪：确认队列配置数据、流转链路数据和进审 / 审核数据可获取 |
| 分析 | 分析输出不合理队列清单和整改建议<br><br>- 业务逻辑不合理：不该建队列、风险程度不足、机审可自动处置、体裁非最小必要<br>- 队列设计不合理：同风险同体裁多队列、同风险跨体裁多队列、无效 / 循环流转、优先级 / 废弃 / 兜底黑盒、非底线待审阻塞、长尾 / 无人认领队列<br>- 判定规则：任一规则命中即判定为不合理队列，按队列去重统计<br>- 输出材料：不合理队列清单、命中规则、涉及风险类型、进审 / 审核数据证据和整改建议 |
| 通知 | 飞书通知人审运营同学，必要时同步治理 BP<br><br>- 发送不合理队列清单、命中规则、数据证据和建议动作<br>- 长周期无有效进审队列建议评估下线<br>- 同风险多队列建议合并或收敛<br>- 体裁不一致问题建议重新确认业务-体裁队列设计<br>- 不自动修改配置，所有变更动作需人工确认 |
| 解决 | 回收队列整改结果，确保不合理队列数归 0<br><br>- 记录处理动作和结论：下线队列、合并队列、修改队列规则、调整体裁映射、补充流转触发 / 终止条件、调整优先级 / 废弃 / 兜底逻辑、公开周知后推动下线<br>- 复查不合理队列数是否归 0，或剩余队列均有明确合理性说明<br>- 沉淀配置变更记录、证据和最终闭环结论 |

## 5. 质量-质检准确率

SOP：
1. 数据口径：质检准确率 = 质检准确量 / 总质检量；通过准确率 = 通过准确量 / 通过质检量；打标准确率 = 打标准确量 / 打标质检量。
2. 目标：整体 95% 以上，涉政底线要求 98%，自营 / X 模式 / BPO / 众包质量目标一致。
3. 质检方式：8 人及以上队列优先交叉质检，小于 8 人队列随机质检，质检方式需为盲检。
4. 质检量：按置信度设定（周度置信、95% 置信区间、误差 ±1%），按单风险域通过 / 打标分别计算。
5. 预警 & 升级：单周不达标升级群组负责人，连续三周不达标升级 CQC 总负责人、人审运营负责人。
6. 质量提升：T+1 完成纠偏，1 周内完成质量提升，通过质检准确率变化、投放准确率等验证效果。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 按照日 / 周时间窗口，抽取质检准确率相关数据<br><br>- 数据维度：质检准确率、通过准确率、打标准确率、质检量、总质检量、风险域、队列、质检方式、供应商模式<br>- 质检方式校验：确认 8 人及以上队列使用交叉质检，小于 8 人队列使用随机质检，质检方式为盲检<br>- 质检量校验：确认质检量满足周度置信、95% 置信区间、误差 ±1% 要求，按单风险域通过 / 打标分别校验<br>- 数据就绪：确认日度 / 周度质检数据已产出 |
| 分析 | 分析输出质检准确率统计和波动归因<br><br>- 指标计算：整体质检准确率、风险域准确率、通过准确率、打标准确率<br>- 目标对比：判断是否低于整体 95% 或涉政底线 98%<br>- 维度拆解：按日度 / 周度、风险域、队列、质检方式、供应商模式拆解<br>- 连续不达标识别：单周不达标、连续三周不达标<br>- 输出材料：统计结果、明细、波动归因、不达标对象清单和整改建议 |
| 通知 | 飞书通知 CQC 群组和人审运营，触发升级条件时上升沟通<br><br>- 单周不达标：通知群组质量负责人、人审运营<br>- 连续三周不达标：升级通知 CQC 总负责人、人审运营负责人<br>- 通知内容包含：不达标指标、目标值、实际准确率、差距、质检量是否满足置信要求、低准风险域 / 队列、波动原因和整改 DDL |
| 解决 | 回收质检准确率结果，确保闭环解决<br><br>- T+1 完成纠偏，记录纠偏动作：SOP 优化、培训、补充质检、调整质检方式、修正质检争议等<br>- 1 周内复查质检准确率和投放准确率是否提升<br>- 连续不达标则按规则升级并持续跟进<br>- 满足动作、证据、复查结果和结论后闭环 |

## 6. 质量-漏放事故监控

SOP：
1. 目标：S01 底线事故人审 0 漏放，S23 非底线事故下降。当前为线下数据。
2. 复盘机制：每个人审 S01 底线事故漏放需 24h 内输出复盘报告。
3. 验收机制：事故发生 1 周内对事故投放验收，S01 漏放率要求为 0，否则复投直至达成。
4. 质检机制：每周对事故定级结果做质检，事故定级准确率要求 95% 以上。
5. 升级机制：miss 目标或有 miss 风险时升级给 CQC 群组负责人及人审运营负责人；明显底线事故漏放升级 CQC 总负责人。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 读取线下事故台账，抽取事故监控相关数据<br><br>- 数据维度：事故 ID、事故等级、风险域、是否人审漏放、S01/S23 标记、事故发生时间、复盘状态、投放验收状态、定级质检结果<br>- 责任人映射：CQC 群组负责人、人审运营负责人、CQC 总负责人<br>- 数据就绪：当前为线下数据，需确认事故台账、复盘记录、投放验收记录、周度质检记录已更新 |
| 分析 | 分析输出事故漏放统计和风险判断<br><br>- S01 监控：统计 S01 人审漏放事故数，判断是否违反清零目标<br>- S23 监控：统计 S23 非底线事故数和趋势，判断是否符合下降目标<br>- 验收检查：识别 1 周内未完成投放验收的事故，S01 漏放率是否归 0<br>- 定级质检：统计事故定级准确率，识别是否 < 95%<br>- 输出材料：事故清单、命中规则、影响等级、原因判断和建议动作 |
| 通知 | 按事故等级和风险域分路径通知，触发升级条件时上升沟通<br><br>- S23 非底线事故或目标 miss 风险：通知 CQC 群组负责人、人审运营负责人<br>- S01 非 N1、LS 等底线事故漏放：升级通知 CQC 群组负责人、人审运营 +1<br>- 明显底线事故漏放（攻击 / 辱骂 / 恶搞 N1、8964 等）：升级通知 CQC 总负责人<br>- 通知内容包含：事故等级、漏放原因、定级质检结果、复盘 DDL、投放验收要求和升级规则 |
| 解决 | 回收事故处理结果，确保闭环解决<br><br>- 24h 内完成复盘报告，记录漏放原因、责任环节、标准优化、审核员培训、质检纠偏<br>- 事故发生 1 周内回收投放验收结果，S01 漏放率必须为 0，否则继续复投直至达成<br>- 每周跟踪事故定级准确率是否 ≥ 95%，发现标准不合理或定级不准确时推动标准优化<br>- 满足复盘、验收、整改、证据和最终结论后闭环 |

## 7. 质量-申诉成功率

SOP：
1. 数据口径：申诉成功率 = 申诉成功作者量 / 申诉作者量（默认按作者口径）。
2. 目标：申诉成功率 < 10%。
3. 数据监控：日 / 周，by 风险域 / 标签监控申诉成功率波动。
4. 归因分析 & 推动优化：拆解申诉成功案例并总结聚类场景，反哺机人审提准、标准尺度优化。
5. 预警及升级：P2 风险域单周不达标推送治理 BP、人审运营；P1 双周不达标推送治理 BP+1、人审运营+1；P0 三周不达标推送治理 BP+2、人审运营+2。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 按照日 / 周时间窗口，抽取申诉成功率相关数据<br><br>- 数据维度：申诉作者量、申诉成功作者量、申诉成功率、风险域、标签、策略、申诉原因<br>- 统计对象确认：默认为作者；如需按视频或账号统计，替换分子分母对象<br>- 责任人映射：治理 BP、人审运营及对应 +1/+2<br>- 数据就绪：确认日度 / 周度申诉数据已产出 |
| 分析 | 分析输出异常风险域清单和归因聚类<br><br>- 指标计算：申诉成功率 = 申诉成功作者量 / 申诉作者量<br>- 目标对比：判断是否超过 < 10% 目标<br>- 连续不达标识别：单周不达标、双周不达标、三周不达标风险域<br>- 原因聚类：拆解标封修改、审核信息差、标准 / case 模糊、机审策略处置、人审执行尺度、申诉执行尺度等原因占比<br>- 输出材料：异常风险域 / 标签 / 策略清单、原因占比、可优化项、预计下降空间和治理建议 |
| 通知 | 飞书通知治理 BP 和人审运营，触发升级条件时上升沟通<br><br>- P2：风险域单周不达标，通知治理 BP、人审运营<br>- P1：风险域双周不达标，通知治理 BP+1、人审运营+1<br>- P0：风险域三周不达标，通知治理 BP+2、人审运营+2<br>- 通知内容包含：申诉成功率、目标差距、连续不达标周数、异常风险域 / 标签 / 策略、主要归因、建议动作和整改 DDL |
| 解决 | 回收申诉成功率优化效果，确保闭环解决<br><br>- 记录治理动作：机审策略提准、人审执行纠偏、标准尺度优化、case 补充、申诉执行口径对齐等<br>- 按 DDL 回收申诉成功率变化，验证是否降至 < 10% 或阶段性目标<br>- 持续不达标则按 P2/P1/P0 规则升级<br>- 满足动作、证据、复查结果和结论后闭环 |

## 8. 质量-质量投放

SOP：
1. 数据口径：投放准确率 = 审核员准确审出 case 量 / 总投放量。当前为线下数据。
2. 投放流程：事故、罚单、指令、LS 重保节点、暑期等特殊时期，将风险 case 投放在队列中，审核结果不生效，2-3 天回收投放准确率。
3. 指标要求：底线风险投放准确率 100%，其他风险要求准确率 95% 以上，如不达标则复投。
4. 升级机制：P2 单次投放准确率不达标推送 CQC 群组质量负责人、人审运营；P1 单次复投不达标或连续两次初投不达标推送 CQC 群组负责人、人审运营+1；P0 连续两次复投不达标或连续三次初投不达标推送 CQC 总负责人、人审运营+2。

| 节点 | 填写内容 |
| --- | --- |
| 感知 | 读取线下投放台账，抽取投放验收相关数据<br><br>- 数据维度：投放批次 ID、风险类型、是否底线风险、队列 ID、总投放量、准确审出 case 量、初投 / 复投状态、回收时间<br>- 责任人映射：CQC 群组质量负责人、人审运营、CQC 群组负责人、CQC 总负责人、人审运营 +1/+2<br>- 数据就绪：当前为线下数据，需确认投放台账、投放 case 明细和回收结果已更新 |
| 分析 | 分析输出投放准确率统计和不达标批次清单<br><br>- 指标计算：投放准确率 = 审核员准确审出 case 量 / 总投放量<br>- 目标对比：底线风险是否达到 100%，其他风险是否达到 95% 以上<br>- 不达标识别：单次投放不达标、单次复投不达标、连续两次初投不达标、连续两次复投不达标、连续三次初投不达标<br>- 输出材料：不达标批次清单、风险类型、队列、准确率差距、错误 case、连续不达标次数和复投建议 |
| 通知 | 按不达标次数和类型分路径通知，触发升级条件时上升沟通<br><br>- P2：单次投放准确率不达标，通知 CQC 群组质量负责人、人审运营<br>- P1：单次复投不达标或连续两次初投不达标，通知 CQC 群组负责人、人审运营 +1<br>- P0：连续两次复投不达标或连续三次初投不达标，通知 CQC 总负责人、人审运营 +2<br>- 通知内容包含：投放批次、风险类型、目标值、实际准确率、错误 case、复投要求和整改 DDL |
| 解决 | 回收投放准确率结果，确保闭环解决<br><br>- 记录处理动作：复投、审核员纠偏、标准澄清、case 讲解、SOP 优化、培训、质检复核<br>- 底线风险必须复查至投放准确率 100%<br>- 其他风险需达到 95% 以上<br>- 复投仍不达标或连续多次初投不达标，按 P1/P0 规则升级<br>- 满足动作、证据、复投结果、最终准确率和结论后闭环 |

## 9. 质量-二确 diff 率

SOP：
1. 数据口径：二确 diff 率 = 二确环节打标与前置队列打标 diff 量 / 总流转二确量。
2. 目标：参考申诉 diff 率测算逻辑，与业务对齐阶段性目标。
3. 数据监控：日度监控二确 diff 率，分析异常风险域、标签、策略等问题。
4. 归因分析 & 推动优化：拆解二确 diff 案例并总结聚类场景，反哺机审人审提准、标准尺度优化。
5. 预警及升级：P2 风险域单周波动 > 10% 推送治理 BP、人审运营；P1 波动 > 20% 推送治理 BP、人审运营；P0 波动 > 30% 推送治理 BP、人审运营。注：需后续明确波动是相对涨幅还是百分点变化。

| 节点 | 填写内容 |
