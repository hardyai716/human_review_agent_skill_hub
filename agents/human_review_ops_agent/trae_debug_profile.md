# TRAE 调试智能体配置

## 调试智能体

- 名称：人审运营智能体
- 模式：debug_only
- 用途：前期验证 Agent 路由、Skill 调用、场景包读取和工具权限边界。

## 启用能力

- 感知 Skill：识别运营模块、指标、任务类型和数据就绪。
- 分析 Skill：生成 QueryPlan、执行只读分析、输出 source_footer。
- 通知 Skill：只生成通知草稿或测试卡片，不发送真实通知。
- 解决 Skill：只记录人工状态、结论、证据和是否继续观察。

## 场景读取策略

优先验证目标态根目录：

```text
references/scenarios/
```

如果 TRAE 跨目录读取不稳定，临时使用 Skill 内调试快照：

```text
skills/{skill_name}/references/scenarios/
```

Skill 内调试快照只能由根目录场景包生成或同步，不允许手工漂移。

## 工具策略

- 默认只读。
- 优先使用 mock Tool 或只读 Tool。
- 真实通知、写状态、更新配置必须人工确认。
- 禁止自动审批、自动改配置、自动触达线上负责人。

## 第一轮样板用例

- 自动处置准确率为什么下降？
- 按策略维度分析自动处置准确率。
- 自动处置准确率预警应该找谁？
- 质检准确率下降了。
- 这个策略怎么了？
