# 验收评估与迭代配置

## 1. 评估目标

第一版以运营专家人工验收为主，评估目标不是证明 Agent 完全自动化，而是判断它是否能稳定减少人工步骤，并保证结论一致、路由正确、证据完整。

核心验收信号：

- 结论一致。
- 路由正确。
- 证据完整。
- 人工步骤减少。

## 2. 验收样例

### 2.1 样例来源

| 来源 | 说明 |
| --- | --- |
| 真实历史问题 | 已有人工处理结论的问题，适合对比 Agent 输出。 |
| 模拟问题 | 运营专家构造的典型问题，适合覆盖流程边界。 |
| SLA 失败事件 | 用于验证催办、升级和闭环回收。 |
| 人工纠错事件 | 用于验证规则和模板是否需要迭代。 |
| 重复问题 | 用于验证历史案例沉淀和迭代触发。 |

### 2.2 样例最小字段

每条验收样例至少包含：

| 字段 | 说明 |
| --- | --- |
| 样例ID | 样例唯一标识。 |
| 输入摘要 | 原始问题、告警、工单或群聊摘要。 |
| 来源类型 | 自然语言、告警、工单、群聊、手工录入。 |
| 期望场景 | 专家判断应命中的业务场景。 |
| 期望问题类型 | 专家判断的问题类型。 |
| 期望 Owner | 正确的数据 Owner 或兜底负责人。 |
| 专家分析结论 | 标准答案，包含影响范围、根因、下一步动作。 |
| 必要证据 | 判断结论必须引用的指标、SOP、样本或历史记录。 |
| 期望状态流转 | 样例应该经历的关键状态。 |
| 人工步骤基线 | 纯人工处理该问题需要的步骤数。 |

## 3. 模块有效性标准

| 模块 | 验收标准 | 判定方式 |
| --- | --- | --- |
| 感知能力 | 事件画像完整，数据就绪等级合理，证据可追溯 | 专家检查事件画像、证据包、缺失字段。 |
| 分析能力 | 影响评估、归因定位、SOP 判定与专家基本一致 | 对比专家结论和 Agent 输出。 |
| 通知能力 | 数据 Owner、通知对象、升级对象正确 | 对比 Owner 表和专家判断。 |
| 解决能力 | 动作、证据、结论三件套完整 | 检查处理结果表和闭环审核。 |
| 编排层 | 快路径、事件流程、SLA、卡点和升级触发正确 | 使用样例回放状态机。 |

## 4. 指标定义

### 4.1 结论一致率

定义：

```text
结论一致率 = 结论被专家判定为一致的样例数 / 已验收样例总数
```

判定口径：

- 影响范围基本一致。
- 主要根因一致或可接受。
- 下一步动作不冲突。
- 未遗漏关键风险。

### 4.2 Owner 路由正确率

定义：

```text
Owner 路由正确率 = Owner 命中正确的样例数 / 已验收样例总数
```

判定口径：

- 主负责人正确。
- 备份负责人合理。
- 升级链路合理。
- Owner 缺失时能进入兜底或人工确认。

### 4.3 证据完整率

定义：

```text
证据完整率 = 关键结论均有证据支撑的样例数 / 已验收样例总数
```

判定口径：

- 影响评估有指标、样本或事实依据。
- 根因判断有支持证据和必要的反证说明。
- SOP 判定引用了规则或流程。
- 关闭事件有处理证据。

### 4.4 人工步骤减少量

定义：

```text
人工步骤减少量 = 人工步骤基线 - Agent 辅助后人工步骤
人工步骤减少率 = 人工步骤减少量 / 人工步骤基线
```

步骤口径：

- 手工查指标算一步。
- 手工整理证据算一步。
- 手工判断 Owner 算一步。
- 手工写通知算一步。
- 手工催办算一步。
- 手工追结论算一步。
- 手工沉淀案例算一步。

### 4.5 SLA 触发正确率

定义：

```text
SLA 触发正确率 = SLA 催办或升级触发正确的样例数 / 应触发 SLA 逻辑的样例数
```

判定口径：

- 响应超时能触发催办或升级。
- 处理超时能触发催办或升级。
- 结论回收超时能触发催办或升级。
- 未超时时不误触发。

## 5. 人工验收流程

```text
选择样例
  -> 准备专家结论和人工步骤基线
  -> 运行 Agent 流程或人工模拟流程
  -> 记录 Agent 输出
  -> 运营专家逐项打标
  -> 统计指标
  -> 标记问题和迭代原因
```

### 5.1 专家打标项

| 打标项 | 可选值 | 说明 |
| --- | --- | --- |
| 场景分类是否正确 | 是/否/部分正确 | 判断是否命中正确业务场景。 |
| 问题类型是否正确 | 是/否/部分正确 | 判断问题分类是否正确。 |
| 结论是否一致 | 是/否/部分一致 | 对比专家结论。 |
| 路由是否正确 | 是/否/部分正确 | 判断 Owner 和升级链路是否正确。 |
| 证据是否完整 | 是/否/部分完整 | 判断关键结论是否有证据。 |
| 状态流转是否正确 | 是/否/部分正确 | 判断是否走了正确状态机路径。 |
| 是否减少人工步骤 | 是/否 | 对比人工步骤基线。 |
| 是否需要迭代 | 是/否 | 判断是否进入迭代候选池。 |

## 6. 处理结果回收

事件关闭前必须回收以下信息：

| 类别 | 字段 | 说明 |
| --- | --- | --- |
| 处理动作 | 处理动作、处理人、处理时间 | 记录实际做了什么、谁做的、什么时候做的。 |
| 根因结论 | 根因、影响范围、最终结论 | 支撑历史案例沉淀和后续复盘。 |
| 证据 | 证据链接、证据摘要、证据是否确认 | 支撑闭环判定。 |
| 异常标记 | 信息不足、Owner错误、分析错误、SLA超时、重复问题、处理无效 | 支撑迭代触发。 |
| 迭代信息 | 是否触发迭代、迭代原因、复盘负责人 | 支撑后续规则、模板或 Skill 更新。 |

非结构化结论处理要求：

- 保留原文引用。
- 提炼标准字段。
- 标记不确定项。
- 标记是否需要人工复核。

## 7. 迭代触发规则

| 触发类型 | 条件 | 迭代对象 |
| --- | --- | --- |
| 人工纠错 | 专家指出分类、分析、路由、SLA 或结论错误 | 场景规则、分析模板、Owner 表、状态机或 Skill。 |
| 重复问题 | 同类问题多次出现 | SOP、案例库、通知模板、自动化规则。 |
| SLA 失败 | 响应、处理或结论回收超时 | SLA 策略、升级链路、通知模板。 |
| 效果退化 | 结论一致率、路由正确率、证据完整率或人工步骤减少量下降 | 对应模块规则或 Skill。 |

## 8. 迭代流程

```text
异常或反馈进入迭代候选池
  -> 运营专家确认是否需要变更
  -> 定位变更对象
  -> 生成变更草案
  -> 人工评审
  -> 发布新版本
  -> 使用验收样例回放
  -> 记录效果对比
```

变更对象包括：

- SOP。
- 流程规则。
- 分析模板。
- 通知模板。
- 数据 Owner。
- SLA 策略。
- 执行权限。
- Skill 实现。

## 9. 回测数据集

### 9.1 数据集字段

| 字段 | 说明 |
| --- | --- |
| sample_id | 样例 ID。 |
| raw_input | 原始输入。 |
| expected_scenario | 期望场景。 |
| expected_problem_type | 期望问题类型。 |
| expected_owner | 期望 Owner。 |
| expected_analysis | 期望分析结论。 |
| required_evidence | 必须引用的证据。 |
| expected_states | 期望状态流转。 |
| expected_closure | 期望闭环结论。 |
| baseline_manual_steps | 人工步骤基线。 |

### 9.2 回测触发

以下情况应触发回测：

- 流程规则变更。
- 分析模板变更。
- Owner 映射变更。
- SLA 策略变更。
- 执行权限变更。
- Skill 实现变更。

### 9.3 回测对比

每次回测应记录：

- 使用的配置版本。
- 使用的 Skill 版本。
- 样例数量。
- 分类一致率。
- Owner 路由正确率。
- 结论一致率。
- 证据完整率。
- SLA 触发正确率。
- 人工步骤减少量。
- 相比上一版本的变化。

## 10. 第一版验收建议

第一版建议先选择 3 到 5 条样例做人工验收，样例应覆盖：

- 一个简单咨询快路径。
- 一个标准事件闭环。
- 一个数据不足需要补信息的事件。
- 一个 Owner 路由和通知事件。
- 一个 SLA 或升级触发事件。

验收通过条件建议：

- 每条样例都能走到明确终态。
- 关键结论都有证据。
- Owner 路由无明显错误。
- 至少能证明部分人工查询、整理、通知或追结论步骤被减少。
