# 分析通用规则

## Aeolus 字段引用快速参考

- 使用 `bytedcli -j aeolus query -r <region> <dataset_id> "<SQL>"` 查询时，Aeolus 会按传入的数据集 ID 编译数据集字段。
- 数据集字段统一写成 `` `[数据集字段名]` ``，例如 `` `[审核准确率]` ``、`` `[队列分类汇总]` ``、`` `[进审量_reviewid]` ``。
- 语义字段写法必须通过 `aeolus query` 携带对应 `<dataset_id>` 执行，由 Aeolus 按数据集 ID 和数据集元信息编译字段。
- 非必要时不手写底层字段逻辑；优先让 Aeolus 语义层编译 `` `[数据集字段名]` ``，保持查询口径和数据集一致。
- 只有在脱离 Aeolus 数据集语义层、语义字段编译失败、需要明确维度归一或需要规避别名解析问题时，才改用物理字段或底层字段逻辑，并在场景文档或 QueryPlan 中说明原因。

## 业务概念字段映射表格式

当场景文档需要说明业务概念在 `aeolus query` 中如何使用时，优先使用以下两种表格格式。

指标类概念保留口径列：

```markdown
| 概念 | aeolus query 使用字段 | 口径 | 说明 |
| --- | --- | --- | --- |
| 审核准确量 | `[审核准确量]` | `[抽检量] - [审核错误量]` | 主指标准确量字段。 |
```

维度、筛选字段、普通数据集字段不写口径列：

```markdown
| 概念 | aeolus query 使用字段 | 说明 |
| --- | --- | --- |
| 日期分区 | `[p_date]` | 数据集字段为 `p_date`，分区字段。 |
| 队列分类汇总 | `[队列分类汇总]` | 通过 `[队列分类 (上游+群组)]`、`[视频质量_队列范围]` 等字段编译而来。 |
```

填写规则：

- `aeolus query 使用字段` 必须填写可直接用于 `aeolus query` 的 `` `[数据集字段名]` `` 写法；字段不存在时写 `无`，不得编造字段。
- `口径` 只用于指标类概念，优先引用数据集标准字段，例如 `[审核准确量] = [抽检量] - [审核错误量]`。
- `说明` 用于补充字段描述、业务含义、默认过滤值、是否主输出维度、是否当前默认 SQL 输出；数据集无字段描述时写 `无` 或简要说明来源。
- `metric_id`、`sub_metric_id`、内部 alias 和脚本变量只属于运行态路由或实现细节，不进入字段映射表。
- 日均、环比、gap 等派生输出不作为数据集字段写入字段清单；如必须说明，放在输出要求或说明中。

## 运行通用规则

- 查询前必须生成 QueryPlan。
- QueryPlan 必须引用命中的单一场景文档。
- 输出必须包含 source_footer。
- 禁止使用未授权、废弃或混淆字段。
- 禁止读取 Skill 外部的 `human_review_ops/references/` 作为运行态口径。
- 每次只加载一个 `references/scenarios/<scenario_key>.md`。
- 用户覆盖样本池、新增维度或要求真实执行时，必须在 QueryPlan 和 source_footer 中标记人工确认。
- 比率类指标必须用分子和分母重算，不得平均或加总比率字段。
- 查询失败、权限失败、字段失败、分区未就绪时停止，不输出业务结论。

    query_plan.md

场景文档是运行态唯一场景资料。公共方法只放跨场景复用规则，不放单一指标口径。
